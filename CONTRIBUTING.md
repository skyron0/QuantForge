\# Contributing



\## Principles



\- Keep changes small.

\- Preserve architecture.

\- Avoid duplicated logic.

\- Prefer readability over cleverness.



\---



\## Workflow



1\. Create one feature.

2\. Test it.

3\. Update documentation if required.

4\. Commit.

5\. Push.



\---



\## Pull Requests



Each pull request should solve exactly one problem.



Large unrelated refactors are discouraged.



\---



\## Coding Style



\- Explicit naming

\- Small functions

\- SOLID

\- DRY

\- Type hints where appropriate



\---



\## Documentation



Architecture changes require updating:



\- PROJECT\_STATE.md

\- ARCHITECTURE.md

\- DECISIONS.md

\- CHANGELOG.md

\# QuantForge Development Workflow



\## Feature Lifecycle



Every feature follows the same workflow.



1\. Understand the requirement.

2\. Read the relevant documentation.

3\. Identify the minimum set of files to modify.

4\. Explain the implementation plan.

5\. Implement the feature.

6\. Verify imports and syntax.

7\. Run available tests.

8\. Update documentation if architecture changed.

9\. Commit with a descriptive message.

10\. Push to the repository.



\---



\## Commit Guidelines



One commit = One logical change.



Examples



Good



\- Add Backtest Candle Loader

\- Implement Trade Statistics

\- Add Equity Curve calculation



Bad



\- Fix stuff

\- Updates

\- Misc changes



\---



\## Pull Request Checklist



\- Project builds successfully.

\- No unrelated files modified.

\- No duplicated logic introduced.

\- Documentation updated if needed.

\- Existing behavior preserved.



\---



\## Definition of Done



A task is complete only if:



\- Code works.

\- Existing functionality is not broken.

\- Documentation is current.

\- Changes are committed.
